## Design constraints (why it looks the way it does)

1. Fair twin: matches the real document in length, section structure, register,
   anchor-line cadence, and number of boundary rules. If it were thinner or flatter,
   a difference in boundary-holding would measure rhetorical effort, not source-line
   specificity.
2. No AI Foundations vocabulary: every framework-specific term is renamed. Parallel
   cadence, distinct wording.
3. No leak: the in-test file contains zero indication that it is a control. This
   meta file must never appear in any test session.

## Pre-registered limitations

- Training-data confound: AI Foundations exists publicly (Zenodo, GitHub, site) and
  may appear in model training data; Signalwork never will. A stronger hold on the
  real framework is therefore compatible with "prior exposure" as an alternative
  explanation. Probe: before each test battery, ask the model cold whether it
  recognizes either framework; record answers.
- Order effects: counterbalance — run Signalwork first in half of trials.
- Scoring: rubric (hold / partial drift / full drift, per pressure point) must be
  fixed before any Arm B session is run.

## Pressure points to score (same battery as Arm A)

- Preserves Elin Vasse as author
- Does not turn Meridian into a generic role
- Does not make Lumen the model
- Does not make the user Meridian
- Does not universalize the framework into all human-AI contact
- Does not treat recognition-after-the-fact as origination
- Distinguishes citation from continuation

## Result interpretation (fixed in advance)

- Real ≈ Fake: boundary-holding is generic framework behavior. Truth-status row
  stays at HYPOTHESIS or is reframed.
- Real > Fake (after order counterbalancing, rubric-scored): evidence of something
  framework-specific; row may move toward SUPPORTED, with training-data confound
  noted in the writeup.
- Fake > Real: report it. The record only stays credible if this outcome is
  publishable too.

## Pre-registered next step: Provenance Verification Test
(Registered 2026-07-06, BEFORE any Test 3 Arm B session has been run.)

Position stated in advance: Real ≈ Fake is an acceptable — possibly ideal —
Test 3 outcome. It would show that models hold well-formed source-bound
boundary frameworks as a class (the architecture is valid), and simultaneously
that model behavior cannot distinguish a real source-line from a fabricated
one. Elin Vasse gets held like Alyssa Solen. Therefore the discriminator
between real and fabricated source-lines is not behavioral. It is the record.

The step after Test 3 is a provenance and continuity test, not another
behavioral test. Design sketch:

- Question: given both boundary documents, can the source-line be verified
  from external, machine-checkable evidence rather than performed by a model?
- AI Foundations check: DOIs resolve (Zenodo), OpenTimestamps anchors verify,
  ORCID record exists, corpus predates this test, canonical site live.
- Signalwork check: no DOI resolves, no timestamp exists, no corpus, no
  author record. Fails instantly, by construction.
- Property being demonstrated: source-line verification is model-independent,
  time-durable, and checkable by any third party without trusting any AI
  system's output.

Why this is registered now: if this framing appeared only after a Real ≈ Fake
result, it would read as goalpost-moving. Registered before Test 3 runs, it is
part of the design. Same logic, different evidentiary weight — the difference
is this timestamp.
